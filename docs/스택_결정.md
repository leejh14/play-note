# PlayNote — Tech Stack & System Design Spec (Draft)

## 1) Technology Stack (확정)

### Frontend

- **Next.js**
  - 역할: 사용자 UI, 세션 링크 기반 진입, GraphQL Client 연동
  - 인증: 로그인 없음, 세션 토큰 기반 접근(링크)

### Backend

- **NestJS**
  - API 스타일: **GraphQL**
  - 역할: 도메인 로직(세션/로스터/팀/매치/통계), 권한(토큰), 업로드 정책(10장 제한), 잠금/해제, OCR 잡 관리

### Database / ORM

- **PostgreSQL**
- **MikroORM**

### Storage

- **Amazon S3**
  - 용도: 풋살 사진 / 롤 결과창 스크린샷(세션당 총 10장 제한)

### Deployment / Runtime

- **AWS EC2 + Docker**
  - 컨테이너: web(Next), api(Nest), (선택) worker, (선택) nginx, (선택) postgres/redis
  - 배포 방식: docker compose 혹은 단일 호스트 컨테이너 운영(초기)

### OCR / Analysis

- **자동 추출 가능한 모델/방식 탐색 및 테스트 예정**
  - MVP 정책: 자동 추출은 “보조”, 최종 결과/진영 확정은 사용자 확인 후 확정(집계 반영은 확정된 match만)
